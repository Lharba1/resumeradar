Features Built & Kept

9 features shipped — each with the reason it made the cut
✓ SHIPPED
CV Upload + Enrichment /upload
User uploads a PDF CV (max 2 MB) and optionally attaches up to 5 "boost" documents — PDF, DOCX, PPTX, or TXT (max 20 MB each). GPT-4o parses the CV into a structured profile. Extra document text is stored as enrichment_context and injected into every downstream AI prompt.
Why it's here
This is the foundation of everything. Without a CV, the optimizer, cover letter, and job scoring have no candidate context. The enrichment feature (PPTX/DOCX portfolio docs) is the key differentiator — it lets the AI pull real project metrics into bullets without the user rewriting their CV manually.

✓ SHIPPED
Job Feed (JobBank + Adzuna) /jobs
Two modes: manual search (enter job title) and CV scan (AI generates 4 queries from the CV profile). Both run JobBank Canada (Cheerio scraper) and Adzuna API in parallel per query. Each job is classified by GPT-4o for visa sponsorship likelihood (high/medium/low), international friendliness, and relocation support.
Why it's here
The core product loop — without jobs, nothing else in the app makes sense. Dual source (JobBank + Adzuna) gives broader coverage. The visa likelihood scoring is the primary differentiator: it filters noise for the target user (international candidates who only care about roles that can sponsor them).

✓ SHIPPED
ATS Resume Optimizer /optimize
Takes a saved CV + a pasted job description. GPT-4o rewrites the CV to maximize ATS keyword match while staying 100% truthful to the source. Returns before/after ATS scores, matched and missing keywords, improvement notes, and a full optimized CV (CVData JSON) that can be downloaded as a PDF.
Why it's here
ATS optimization is the main monetizable feature — the #1 pain point for international candidates who get zero callbacks despite being qualified. The strict bullet-count rule (4–6 per role) and keyword injection make the output genuinely usable, not just reordered text.

Cover Letter Generator /cover-letter
Input: CV + job description text or a job URL (server-side fetch + HTML strip). Language toggle: Canadian English or Quebec French. Output: 3–4 paragraph cover letter with proper letterhead — subject line, greeting, paragraphs, closing, contact info. Can be copied as text or downloaded as a PDF.
Why it's here
Completes the application package. The URL fetch feature is critical — most users find jobs on LinkedIn or Indeed and shouldn't have to copy-paste the full description. The bilingual output (EN/FR) is uniquely Canadian and covers Quebec employers who require French.

✓ SHIPPED
Interview Prep Coach /interview
Generates 12 tailored interview questions (4 behavioral, 4 technical, 2 situational, 2 Canadian culture) from a job description + optional CV. Each question includes a practice area where the user types their STAR answer and receives AI feedback: score (1–10), STAR breakdown (strong/weak/missing per element), strengths, improvements, and a model rewrite.
Why it's here
Completes the full hiring funnel. This feature has zero data accuracy risk (unlike salary or immigration data) and high perceived value. The Canadian culture questions address a real pain point for immigrants who aren't familiar with Canadian workplace communication norms. Low liability, high stickiness.

Application Tracker /tracker
Kanban board with 6 columns: saved → applied → screening → interview → offer → rejected. Drag-and-drop via @dnd-kit. Notes field per application, applied_at timestamp. Full job details visible in each card.
Why it's here
Retention anchor. Without a tracker, users apply once and leave. With it, they keep coming back to update statuses, which feeds the Dashboard analytics and creates a habit loop. It's also the simplest feature to build and one of the most-requested by job seekers.

Analytics Dashboard /dashboard
Metrics: total applications, applied count, response rate, avg ATS score, avg relocation score. Visa distribution chart (Recharts). GPT-4o generates career insights only when the user has at least 1 application — best countries, best job types, improvement tips.
Why it's here
Closes the feedback loop. Users who see their response rate and ATS trends stay engaged longer. The AI insights are gated behind real data (min 1 application) to avoid empty advice — a deliberate design decision to keep quality high.

Generates a full ATS-optimized Canadian CV from a CV profile + target job descriptions. Detects whether to produce English, French, or both based on job description language signals. Features Cut — and Why

Planned features that were explicitly removed. These are not "coming soon" — they were cut for hard reasons.
⚠️
Cut ≠ Failed. Cutting a feature before it ships is good engineering. The features below were cut because shipping them as-scoped would have created legal liability, destroyed user trust through inaccuracy, or required data sources that don't exist. Each cut makes the product sharper.
✂ CUT
Express Entry CRS Score Estimator
Originally planned as part of the "Immigration Intelligence Layer": the user inputs their profile and receives a Comprehensive Ranking System (CRS) score estimate used for Canada's Express Entry immigration program.
Why it was cut — Legal liability
Under Canadian law (IRCC regulations), only Regulated Canadian Immigration Consultants (RCICs) and immigration lawyers are permitted to provide immigration advice. "Your CRS score is 423 and you qualify for Express Entry" is immigration advice, even if prefaced with a disclaimer. If a user acts on a wrong estimate and misses a filing window or enters the wrong Express Entry pool, the platform faces real legal exposure. No disclaimer negates this under IRCC's framework — the advice is the problem, not the framing.

What we kept instead: A NOC code display (informational, not advisory) with a hard callout: "Consult a Regulated Canadian Immigration Consultant (RCIC) for immigration decisions."
✂ CUT
"Sponsorship Likely" Job Tagging
Jobs in the feed were to be tagged with a "Sponsorship likely" badge based on AI analysis of company size, industry, job description keywords, and employer history with work permits.
Why it was cut — Accuracy < 60%, trust destroyed
There is no reliable signal for whether a specific employer will sponsor a work permit for a specific role. Company size correlates weakly. The absence of "visa sponsorship" in a job posting doesn't mean they won't — and its presence doesn't mean they will. An AI making this call from job description text alone would be wrong roughly 40–50% of the time. A badge that's wrong half the time is worse than no badge: users apply to jobs expecting sponsorship and get rejected at the offer stage. That experience destroys trust in the entire product.

What we kept instead: The existing high/medium/low classification based strictly on what is explicitly stated in the job description (high = explicitly mentions visa/work permit, medium = multinational, low = unclear).
✂ CUT
Provincial Nominee Program (PNP) Eligibility Checker
Would scan the user's profile and show which provincial streams they might be eligible for across all 13 Canadian provinces and territories.
Why it was cut — Maintenance cost + legal risk
PNP eligibility rules change frequently — some provinces update streams monthly. Ontario's Human Capital Priorities stream, BC PNP Tech, Alberta Opportunity Stream: each has different occupation lists, score thresholds, language requirements, and intake periods that open and close unpredictably. Building this accurately would require a team monitoring 13 provincial immigration websites continuously and updating a rules engine. As a solo developer, maintaining stale PNP data creates the same liability as the CRS estimator — users make decisions based on outdated information.

Features Deferred — Later, Not Never

These were planned but need specific prerequisites before they can ship correctly.
⏳ DEFERRED
Job Alerts & Email Digest
Weekly/daily email of new matching jobs. Originally scoped for 1–2 days but revised after premortem identified hidden complexity.
Why it's deferred — Wrong scope estimate
"Match your CV" requires a skill-index pipeline that doesn't exist. CASL (Canada's anti-spam law) requires a verifiable one-click unsubscribe or the sender faces fines. Resend will suspend accounts with high unsubscribe rates.

Prerequisite to ship: Build keyword-only matching first (v1 = match job title to user's last search), add unsubscribe endpoint, configure Resend with proper SPF/DKIM for the domain.
⏳ DEFERRED
Salary Intelligence
Province-by-province salary bands by NOC code. Cost-of-living comparison. Auto-inject salary range into cover letters.
Why it's deferred — No reliable data source
AI-estimated salary data will be wrong for niche roles and small provinces. Auto-injecting wrong salary expectations into a cover letter makes the candidate look uninformed — worse than not having the feature.

Prerequisite to ship: Integrate StatsCan Job Vacancy and Wage Survey data (annual CSV, publicly available) mapped to NOC 2021 codes. Only then should salary data appear in the UI.
⏳ DEFERRED
NOC Code Matcher
Map the user's job title and skills to the correct NOC 2021 code (National Occupational Classification). Display the code with a link to ESDC's official description.
Why it's deferred — Accuracy requirement
GPT-4o hallucinating a wrong NOC code is a real risk — "Software Engineer" and "Software Developer" map to different codes in NOC 2021, and the distinction matters for immigration applications.

Prerequisite to ship: Load ESDC's official NOC 2021 dataset into a vector store and use semantic search against canonical descriptions — not prompt-only classification. Add a hard disclaimer: "Informational only — verify with an RCIC before using in any application."
⏳ DEFERRED
LinkedIn Profile Optimizer
Paste LinkedIn About/Experience → AI rewrites for Canadian ATS. Headline + summary generator.
Why it's deferred — Scope, not risk
Technically straightforward (same flow as ATS optimizer). Deferred because the existing CV + cover letter + interview prep pipeline is more impactful for the current user.

When to ship: After job alerts. LinkedIn optimizer is a Tier 2 feature — valuable but not on the critical path for getting a Canadian job offer.
⏳ DEFERRED
GC Jobs Integration
Government of Canada jobs feed (jobs.gc.ca). Federal jobs are major for immigrants — stability + PR pathways.
Why it's deferred — Scraper complexity
GC Jobs has anti-scraping measures and no public API. A Cheerio scraper is possible but fragile. The current JobBank scraper is already fragile — adding a second fragile scraper doubles maintenance.

When to ship: After stabilizing JobBank scraper with a fallback (retry + selector versioning) and validating user demand for federal jobs specifically.

Premortem Findings

We ran a premortem on the original 4-feature roadmap. Here's what we found and how we responded.
💡
A premortem asks: "It's 3 months from now and this roadmap failed. Work backwards — what killed it?" Running this before writing code saved us from 3 significant mistakes.
HIGH RISK
Job Alerts — "1–2 days" was wrong by a factor of 3
🔥 What would have killed it
The "match your CV" alert requires a skill-index pipeline that doesn't exist. Without it, users get irrelevant emails → high unsubscribe rate → Resend suspends the account for spam signals. Also: CASL (Canada's Anti-Spam Law) requires a verifiable one-click unsubscribe in every commercial email — missing this is a $1M CAD fine risk.
✓ What we did instead
Deferred to after the core funnel is proven. When built: v1 will use keyword-only matching (no CV skill index), include a proper unsubscribe endpoint, and configure Resend with domain SPF/DKIM before sending a single email.
HIGH RISK — LEGAL
CRS Estimator — immigration advice without an RCIC license
🔥 What would have killed it
IRCC explicitly prohibits non-RCICs from giving immigration advice. "Your CRS score is 423" is immigration advice regardless of the disclaimer. Wrong estimate + user misses filing window = platform liability. Canadian courts have ruled on this. This is not hypothetical.
✓ What we did instead
Cut the CRS estimator entirely. When the NOC matcher ships, it will display the code as purely informational with a hard callout: "Not immigration advice. Consult a Regulated Canadian Immigration Consultant (RCIC) for any immigration decisions."
HIGH RISK — DATA
Salary Intelligence — no real data source exists for free
🔥 What would have killed it
AI-estimated salary bands by NOC and province will be confidently wrong for niche roles and small provinces. Auto-injecting wrong salary expectations into a cover letter makes the candidate look uninformed in interviews. This is worse than the feature not existing at all.
✓ What we did instead
Deferred until StatsCan Job Vacancy and Wage Survey data (publicly available annual CSVs) is properly loaded and indexed by NOC 2021 code. Only then should a salary estimate appear.
MEDIUM RISK
Interview Prep — one-shot feature with no retention loop
⚠ What would have degraded it
Without saving sessions or tracking progress across questions, users run it once, get generic questions, and never return. Overestimated stickiness.
✓ What we did instead
Built per-question answer + feedback persistence in React state (answers and feedbacks indexed by question). User can practice multiple questions in one session without losing progress when switching between them. Future v2: persist sessions to Supabase so users can resume practice across sessions.

Key Technical Decisions
html2pdf.js style isolation: <style> tags in innerHTML bleed into the browser's global scope, making PDF columns render full-width. Fix: zero <style> blocks in PDF templates, all inline styles on a #cv-root div, capture container.firstElementChild (the 210mm div) not the wrapper, set html2canvas: { width: 794 }.
page-break-inside: avoid scope: Applying this to sections (not entries) causes entire sections to be pushed to page 2, creating blank pages. Only apply to individual entry cards (per job, per education row).
enrichment_context flow: Extra docs (PPTX/DOCX/PDF/TXT) are parsed server-side and stored in cv_profiles.enrichment_context (max 40k chars). This column is injected as "PROJECT & PORTFOLIO CONTEXT" into every downstream AI prompt — optimizer, cover letter, and interview prep.
Rate limit bypass in dev: checkAndIncrement() returns { allowed: true, remaining: 999 } when NODE_ENV === "development". No real API usage is counted during local development.
bodySizeLimit: "25mb": Set in next.config.ts experimental.serverActions to handle large PPTX/PDF uploads. The default 1MB limit silently rejected files, which manifested as a cryptic parse error.

Key Engineering Decisions

Non-obvious choices and the reasoning behind them
Single PDF template — inline styles only
The cvPdf.ts template returns a raw <div> with no <style> block. All 40+ style properties are inlined on every element.
Decision
html2pdf.js uses html2canvas to rasterize DOM. When you inject a <style> block via innerHTML, the browser applies those styles globally — so body gets width: 210mm, breaking the page layout and clipping the right column. Inline-only styles are the only reliable escape. Combined with capturing container.firstElementChild (not the wrapper) and html2canvas width: 794, this produces pixel-accurate A4 output.
GPT-4o for all AI tasks — no Haiku/Sonnet split
Every AI feature uses GPT-4o with response_format: json_object. No model routing by task complexity.
Decision
CV optimization and cover letter generation require high output quality — a downgrade to a smaller model produces visibly worse bullet quality and keyword injection. The rate limits (5–15/day per feature) keep costs manageable at scale. json_object mode eliminates JSON parsing errors entirely — the model is forced to return valid JSON or the call fails cleanly.
enrichment_context stored at parse time, injected at use time
Extra documents are parsed and stored in the DB at upload. Each AI feature re-reads this column and injects it into the prompt.
Decision
Alternative: re-parse docs on each AI call. This would mean re-uploading files, hitting 25MB limits on every optimize call, and adding latency. Storing processed text solves all three. The 40k char limit on enrichment_context prevents token budget exhaustion in downstream prompts.
Fail-open rate limiting
If the rate limit RPC itself fails (network error, Supabase timeout), the call is allowed through with remaining: 1.
Decision
Failing closed (blocking the user when rate limit check fails) would cause false rejections during Supabase cold starts or edge timeouts. The cost of one extra AI call leaking through is lower than blocking a legitimate user. Rate limits are a soft guardrail, not a security boundary.

Known Limitations & Tech Debt

Honest list of what's imperfect — and whether it matters
Stale supabase-schema.sql
The committed schema file doesn't include enrichment_context, user_id columns, or the api_usage rate limit table. Live DB was migrated via Supabase MCP.
Impact: Medium
A fresh deployment from the SQL file alone would fail. A new dev would need to run the MCP migration history manually. Fix: regenerate schema from Supabase dashboard and commit it.
JobBank scraper is fragile
Cheerio selectors against Job Bank Canada's HTML. Any site redesign will break the scraper — silently (returns empty results, not an error).
Impact: High if it breaks
Mitigate with a selector version file and an admin alert if the result count drops to 0 for 3 consecutive fetches. JobBank has a public API in beta — worth monitoring.
No CV update or delete
cv_profiles only supports insert. Users accumulate CVs with no way to remove old ones from the UI.
Impact: Low now, medium at scale
A user with 10 CVs in the dropdown is a UX problem. Fix: add a delete button in the CV panel with a confirmation dialog. Also enable update (re-parse with a new PDF, same cv_id).
No error monitoring
No Sentry, no server-side log aggregation beyond console.error. Production failures are invisible unless a user reports them.
Impact: High in production
Add Sentry (free tier) with Next.js plugin. Track: AI call failures, scraper empty results, rate limit hits (to validate the limits are set correctly), and PDF generation errors.
@anthropic-ai/sdk installed but unused
The package is in dependencies but no route uses it. Leftover from an earlier exploration.
Impact: Negligible
Adds ~1.5MB to the bundle. Run npm uninstall @anthropic-ai/sdk when convenient.
Adzuna visa filter not implemented
The visaOnly parameter is accepted but ignored in the Adzuna integration — the API doesn't expose a visa filter.
Impact: Medium on UX
Adzuna results don't respect the "visa only" toggle. The GPT-4o classification still scores them correctly — it just means more "low" results appear in the feed when visaOnly is checked. No silent data quality issue, just UX noise.
